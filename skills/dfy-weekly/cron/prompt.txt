Run the /dfy-weekly skill for Joy for the current week.

The week runs Friday → Thursday. The target week is the **most recently COMPLETED** Fri→Thu week — i.e. it ends on the most recent Thursday that is strictly before today, and starts on the Friday 6 days before that Thursday.

This cron normally fires on Friday, so the target week is last Friday → yesterday (Thursday). But do NOT hardcode "today is Friday" — compute it from today's actual date so a late/catch-up run (Sat/Sun) still reports the same completed week:
- endDate = the most recent Thursday strictly before today
- startDate = that Thursday minus 6 days (the Friday)
Example: if today is Fri 2026-06-05, the completed week is Fri 2026-05-30 → Thu 2026-06-04. If today is Fri 2026-06-12, it's Fri 2026-06-05 → Thu 2026-06-11.

Follow skills/dfy-weekly/SKILL.md exactly:
- Read AVD_TICKET_API_KEY from CSL/.env
- Fetch tickets from https://avada-ts-a9cb0.web.app/api/external/tickets/by-date with appName="JOY Loyalty", startDate, endDate
- Fetch tags from https://avada-ts-a9cb0.web.app/api/external/tags
- Identify DFY tickets (subject starts with [DFY]), determine CS creator (members[].isCreate), apply the open-only / sale_request / liz-no-tag filters
- ALSO fetch the previous Fri→Thu week (the 7 days before startDate) with the same filters, so the Overview can show week-over-week deltas.
- Build the 3-line Overview block WITH compare vs last week:
  * Tổng ticket: {this} ▲/▼ {delta} _(tuần trước {prev})_
  * Adopted: {this} ({pct}%) ▲/▼ {delta_pct}% _(tuần trước {prev}, {prev_pct}%)_   — delta on the percentage
  * Ticket không có tag adopted: {this} ▲/▼ {delta} _(tuần trước {prev})_
- Build per-CS breakdown tables: Date | Ticket | Store | Tasks | Tags  (NO Point column, NO Status column)
- End the report with a "---" divider then a "## Note" section containing the placeholder "_(Liz điền — feedback khi review với CS)_".
- Write the markdown to a temp file /tmp/joy-dfy-{YYYY-W##}.md (ISO week of the Thursday end date). Do NOT start the body with an "# Joy DFY - ..." H1 — the page title already shows that. The body must start directly at "## Overview".
