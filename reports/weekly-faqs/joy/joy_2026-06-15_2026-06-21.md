## 1. Getting Started & Setup

### Q1: How do I launch my loyalty program / can you review my setup before I go live? 🔁 recurring
**Frequency:** ~14 sessions
**Standard answer:**
Core steps: (1) set your earning + redeeming rules under **Joy Admin → Reward programs**; (2) build a loyalty page via **Online Store → Themes → Customize → Add section → Joy Loyalty**; (3) enable the storefront widget in **Theme Editor → App embeds → Joy Loyalty - Widget → Save**; (4) make it visible from your menu/home page and promote it. A solid starter setup: a signup bonus, 1 point per $1, and a 100-point = small-discount reward. Loyalty needs **Shopify Admin → Settings → Customer accounts** enabled so customers can sign up and see their balance. If you'd like, share collaborator access and we'll review your configuration and flag anything before you go live.

### Q2: Can I rename "Points" or the loyalty program to my own brand term? 🔁 recurring
**Frequency:** ~4 sessions
**Standard answer:**
Yes. Rename the points currency at **Joy Admin → Settings → General → Custom point label** (enter singular/plural, e.g. "Beans"). To rename the program itself (e.g. "Club Joocy", "Programa de Puntos") edit the program name in **Reward programs**. You can also set a custom discount-code prefix (must follow the `ABCD-` pattern) under **Settings → General**.

## 2. Testing & Going Live (Sandbox / Test mode) 🆕

### Q3: How do I test or preview the program without making it public or emailing customers? 🆕 new
**Frequency:** ~8 sessions
**Standard answer:**
Two safe ways to test:
1. **Sandbox / Test mode** — toggle the program's live switch off (Sandbox) at the top of the reward program. The program keeps running for listed test accounts only, so you can verify earning/redeeming without exposing it to all shoppers. Note: with Sandbox on and the app embed off, the floating widget will **not** appear publicly — that's expected.
2. **Draft/duplicate theme** — enable the Joy widget on a duplicated (unpublished) theme and preview there, leaving your live theme untouched.

Important: while testing, keep **notifications off** (see Q5) so test activity doesn't email real customers. After testing, switch the program back to live. We've seen cases where a teammate flipped a store out of Sandbox and customers received a burst of emails — so always re-confirm Sandbox + notification state before and after edits.

### Q4: Does the program still track points on the Free/Starter and Essential plans? 🔁 recurring
**Frequency:** ~5 sessions
**Standard answer:**
Yes. Customers still earn and hold points, and can redeem, on Free/Starter and Essential — the plan mainly changes the order quota and which advanced features (VIP Tiers, Milestones, POS, etc.) are available, not whether points are tracked. When you cross your plan's monthly order quota you can upgrade to the next tier at any time.

## 3. Points — Earning

### Q5: Why weren't signup / welcome bonus points awarded to a customer? 🔁 recurring
**Frequency:** ~10 sessions
**Standard answer:**
Check that (a) the customer actually completed the signup action tied to the bonus, (b) they have/were logged into a customer account with that email, and (c) the program was **live** (not Sandbox) at the time. If everything looks correct but points are missing, you can add them manually at **Joy Admin → Customers → [customer] → adjust points**. For repeat or unexplained misses (e.g. several customers not getting the welcome bonus), share the customer emails and we'll escalate to the technical team to investigate.

### Q6: Why did an order earn 0 points / award points only when "Delivered"? 🔁 recurring
**Frequency:** ~9 sessions
**Standard answer:**
Points are awarded when the order reaches the trigger you set at **Joy Admin → Settings → Order → Reward by order status** (**Paid** or **Fulfilled**). Common causes of 0 points: the order hasn't reached that status yet (e.g. COD/unpaid, or paid-but-unfulfilled when set to Fulfilled), the customer had no account at purchase, or retroactively imported orders that came in at 0. Joy does not have a native "on Delivered" or "30 days after delivered" trigger; the closest options are awarding on Fulfilled plus **Pending Points** (Advanced/Ultimate, **Settings → Additional Features → Pending Points**) to hold points for a waiting period before they become spendable. For retro orders that imported at 0 points, we can help re-trigger the earning so balances backfill correctly.

### Q7: Can I earn points only on full-price products and exclude sale/discounted items? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Yes — scope the earning rule (and the redemption rule) to a collection. The cleanest way: keep all sale items in one collection (e.g. a "SALE" collection tagged `sale`) and set the **Place order** earning rule's product condition to **exclude** that collection, so customers earn points only on full-price items. To also stop them redeeming on sale items, apply the same collection restriction on the redeeming program's **Applicable categories**. This prevents discount stacking on already-discounted products.

### Q8: Can different VIP tiers earn points at different rates (1x / 1.5x / 2x)? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Yes, on **Advanced/Ultimate**. Use **VIP Tiers** (Joy Admin → Membership → VIP Tiers): each tier can carry a points-earning **perk/multiplier** so, e.g., Bronze earns 1 pt/$1, Silver 1.5, Gold 2. The storefront shows the rate matching the logged-in customer's tier. If a multi-tier store sees only one earning rate displayed, confirm the perk is configured per tier and that you're testing with an account in the right tier (a known display mismatch where the popup value didn't match the tier setting was logged to the tech team).

### Q9: Do automatic subscription renewal orders (Seal, BOLD, Subi) earn points? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Shopify doesn't natively pass subscription-renewal data to apps, so renewals need a **Shopify Flow** bridge. Connect Shopify Flow at **Joy Admin → Settings → Integrations**, then import Joy's **"Joy Loyalty - Subscription Order Data Sync.flow"** template at **Shopify Admin → Apps → Shopify Flow → Import** and turn it on — after that, renewal orders can earn points like one-time orders. Set the renewal earning rule to the same rate as one-time purchases; because the one-time rule is scoped to one-time purchases, points won't double on the initial order. If a specific subscription app still doesn't connect or renewals aren't earning, share the order number and we'll escalate.

### Q10: Daily-visit / social / Google-review points aren't being awarded. 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
- **Visit website / daily visit** (All plans) and **Social activity** (Essential+) are trust-based — the customer must be logged in when they perform the action; if a logged-in test account isn't getting the daily points, re-test on a fresh session and share the account email so we can check.
- **Google reviews** (Essential+) require **admin approval** before points are granted — pending reviews appear in Joy for you to approve; if your usual approval list disappeared, tell us and we'll locate it.
- **Instagram-specific** programs (Advanced/Ultimate) need an IG Business/Creator account connected under **Joy Admin → Integrations → Instagram**, and the customer must link their IG username.
If an action is configured and the customer still gets nothing, send the email + activity and we'll investigate.

## 4. Points — Redemption & Discount Codes

### Q11: A customer can't redeem / a redeemed code is invalid or errors at checkout. 🔁 recurring
**Frequency:** ~8 sessions
**Standard answer:**
Joy redemptions create standard Shopify one-time-use codes. If a code is rejected, check it hasn't already been used or expired, and that the cart meets the redemption rule's conditions (minimum spend, eligible products/collections). Also confirm the customer is logged in and redeeming from their own account. For codes erroring on a valid cart, share the customer email + code and we'll escalate. Tip for customers who "can't find where to redeem": the flow is redeem in the widget first, then the code applies at checkout — see Q12 to make this more intuitive.

### Q12: Can points be redeemed directly in the cart drawer / at checkout instead of pasting a code? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
- **Cart drawer / cart page redemption** (the "Redeem in line" block) is available on **Advanced+** and is the recommended setup for non-Plus stores — we can add the block to your cart drawer or cart page so customers redeem right before checkout.
- **Redeem directly on the Shopify checkout page** requires **Shopify Plus + Checkout Extensibility** and Joy **Ultimate** (Advanced can request via sales).
You can also set **Settings → General → Redemption behavior** so applying a discount opens the cart drawer automatically, reducing the "redeem then re-find checkout" confusion customers report.

### Q13: Can a Joy points discount be used together with another coupon (stacking)? 🔁 recurring
**Frequency:** ~4 sessions
**Standard answer:**
Yes, if combinations are enabled. On the redeeming program, set the **Combinations** option so the generated code can stack with the discount classes you allow (order/product/shipping). Both the Joy redemption rule and the other Shopify discount must permit combining. After updating, test with a real cart to confirm both apply.

### Q14: How do I sort the redemption options on the loyalty page? 🆕 new
**Frequency:** ~1 session
**Standard answer:**
The order of redeem options on the loyalty page follows their arrangement in your redeeming programs — we can reorder them (e.g. lowest to highest points) for you, or you can drag/reorder the programs. Tell us the order you want and we'll set it.

## 5. Migration & Importing Data

### Q15: How do I migrate from another loyalty app (Smile, One Loyalty, PrestaShop, WooCommerce, etc.)? 🔁 recurring
**Frequency:** ~14 sessions
**Standard answer:**
Two paths: **Import** (self-serve, **Joy Admin → Customers → Import** CSV — points balance + birthday) and **Migrate / assisted** (full data incl. VIP tier, status, phone). For Smile, look for the Smile migration guide; if you don't see a "Migration" wording on the page, contact support and we'll enable the assisted flow and map your data. For a CSV, provide customer email + point balance (and tier, if importing tiers). Large customer files (>10 MB) can't go through chat — share a Google Drive/Dropbox link instead. For high-volume stores (10k+ orders) considering Ultimate, the Sales team can discuss a custom quota/discount.

### Q16: My import failed — "Non-existent customers" or orders imported at 0 points. 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
- **"Non-existent customers":** the import matches on the customer email already existing in **Shopify**. Confirm the email in your CSV exactly matches the Shopify customer record (same email, no typos/extra spaces); customers must exist in Shopify first. Send us the file and we'll help re-run it.
- **Imported orders at 0 points:** retroactive orders can land at 0 if the earning rule wasn't applied. We can delete and re-add the points with the correct rule so the right balance (and tier) lands — just confirm the date range (e.g. last 12 months) and which customers already had manual points so we don't double-credit.

### Q17: How do I export my Joy customers / points (and import tiers back)? 🔁 recurring
**Frequency:** ~5 sessions
**Standard answer:**
Export from **Joy Admin → Customers → Export** (if you don't see the Export button, ask support to enable it, then refresh). By default the export may not include the **VIP tier** column — we can enable tier in the export. If you re-import the same file, it **updates** existing customers (matched by email) rather than creating duplicates, so you don't need to delete the current list first. To bulk-update only tiers without changing points, import a tier-only file. Full programmatic API export is an Ultimate feature.

### Q18: Can I bulk add points to all/many customers (and bulk issue discounts / assign tiers)? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Yes. The quickest way to give existing customers a bonus is a **CSV bulk adjustment** — add a fixed amount (e.g. 100–200 pts) to a list of customers at once via the Customers import (the file maps each customer's email to the points to add). For bulk **exclusive-tier** assignment or bulk discounts to a specific list, share the customer list and the rule (e.g. "$20 off over $110") and we'll help set it up. Note bulk-adding points can move customers across VIP tiers if tiers are points-based.

## 6. VIP Tiers

### Q19: How do I set up VIP tiers, tier perks (free shipping / % off), and entry rewards? 🔁 recurring
**Frequency:** ~9 sessions
**Standard answer:**
VIP Tiers are on **Advanced/Ultimate** at **Joy Admin → Membership → VIP Tiers**. Pick a calculation rule (Points earned / Amount spent / Number of orders). For a **one-time** reward when a customer reaches a tier, use **Entry reward**; for an **ongoing** perk on every qualifying order (e.g. Gold always gets free shipping, a standing % off, or extra earning rate) use **Tier privileges → Add perks**. You can give different perk counts per tier (e.g. Tier 1 = 1 perk, Tier 3 = 3 free-shipping vouchers). Remember to **Launch** the tier after editing — a perk/coupon "not appearing" is usually an un-relaunched tier or a reward placed under the wrong section.

### Q20: A tier perk discount isn't applying (online or POS) / a customer got a tier reward twice. 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
If a tier privilege (e.g. automatic per-tier discount) isn't applying for a logged-in customer at checkout or in Shopify POS, first re-confirm the perk is configured and the tier was **relaunched**. If it's set correctly and still doesn't register on a live test account across web and POS, share the exact flow (logged-in user → cart → checkout, discount missing) and we'll escalate to the technical team. For a customer who received the **same tier reward twice** (and may have used both), send the customer email and order so we can investigate whether others were affected and remediate.

### Q21: Can a customer keep their tier when they redeem/spend points, and can I manually set a tier? 🔁 recurring
**Frequency:** ~3 sessions
**Standard answer:**
For **points-based** tiers, spending points can lower a balance and affect tier position; if you want spending not to demote customers, base tiers on **Amount spent** (cumulative spend isn't reduced by redemptions) or use an **Exclusive tier** for protected groups. To assign a customer manually (ambassadors, wholesale, or "keep at top tier regardless of points"), create an **Exclusive tier** (Advanced/Ultimate) and assign via **Joy Admin → Customers → [name] → Assign to exclusive tier**; you can also bulk-assign from a list.

## 7. Referral

### Q22: My referral isn't working — code invalid, no discount applied, or "test mode," and when is the referrer rewarded? 🔁 recurring
**Frequency:** ~7 sessions
**Standard answer:**
Key points: the friend must use the **referral link** and typically a **new** email/account; if they check out with a different email than the one referred, attribution breaks. A "running in test mode" message means the program is in Sandbox/preview — go live for real rewards. On the same PC+browser where the merchant is logged into their own store, the referral page may just show the code without prompting for a friend's email; testing in incognito or on a phone/another account behaves correctly. The **referrer** is rewarded **after the friend completes a qualifying order**, not on link click — and you can disable the referrer's reward if you only want to reward the friend. Referral limits scale by plan (Starter 30 / Essential 100 / Advanced+ Unlimited).

## 8. Store Credit vs Points

### Q23: How does store credit work and why didn't my test order get credited? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Store credit grants a real cash-value balance on the customer's Shopify account, applied **automatically at checkout** (no coupons) — it's powered by Shopify's native store credit. Enable it by selecting **Store Credit** as the reward type in a supported program; the first time, click **Grant Access → Confirm** to authorize Joy to manage Shopify store-credit balances. If a test order didn't credit, the most common cause is the order wasn't marked **Paid** (credit awards on the configured order status). "Failed to adjust store credit" errors usually mean the access grant or a Shopify permission is missing — re-grant access; if it persists, send us the customer/order and we'll escalate.

## 9. Customer Accounts, Duplicates & Merge

### Q24: A customer has duplicate accounts / points split across two profiles — can you merge them? 🔁 recurring
**Frequency:** ~4 sessions
**Standard answer:**
This happens when a shopper checks out under two emails or across channels (e.g. web + TikTok shop). Send us both customer emails and the surviving account; support can merge the profiles and consolidate the point balance. If you merge the customers natively in Shopify, the points are combined accordingly. Encourage customers to always log in before checkout to avoid splits. For recurring duplicates on one store, we can investigate the root cause with the tech team.

### Q25: A customer added in-store (POS) needs to access the same loyalty account online but has no password. 🆕 new
**Frequency:** ~2 sessions
**Standard answer:**
With New Customer Accounts (SSO), the customer is automatically recognized as a member once Shopify identifies them by email — they don't set a traditional password; they log in via the email/one-time-code flow on your storefront and will see the same loyalty balance you created at the counter. With Legacy accounts they register once in the store. Either way, the in-store and online activity tie to the same email.

## 10. Birthday Rewards

### Q26: How do birthday rewards work, why hasn't a customer's fired, and can they choose the gift? 🔁 recurring
**Frequency:** ~3 sessions
**Standard answer:**
Birthday rewards fire on the customer's stored birth date and only if the birthday exists **before** that date passes — a birthday added the day after won't backfill that year (so a customer who just entered their birthday will only see it work next cycle). Make sure the customer captured their birthday (POS can't capture it — customers add it in their Joy profile online). If you offer several birthday gift options, the default lets the customer pick **one** from the set; turn on the option to let them choose. Enable the birthday email and customize the sender name/text under **Joy Admin → Settings → Notifications**.

## 11. Transaction / Order Limits

### Q27: I hit a transaction limit but have few orders — what is it and can I get unlimited? 🔁 recurring
**Frequency:** ~6 sessions
**Standard answer:**
A **transaction in Joy isn't a Shopify order** — 1 transaction = 1 loyalty activity (earning, redeeming, referral, etc.), so a single order can create several transactions and the count climbs faster than your order count. If you've hit the transaction limit while still inside your plan's order quota, support can enable **Unlimited transactions** for verified stores — share your store URL and we'll escalate to the team to clear the warning.

## 12. Pricing, Plans & Refunds

### Q28: How much does Joy cost, what's on each plan, and are there discounts / longer trials? 🔁 recurring
**Frequency:** ~10 sessions
**Standard answer:**
Starter $0 (250 free orders/mo), Essential $29/mo (500 orders), Advanced $129/mo (2,000 orders), Ultimate $499/mo (7,000 orders), with per-100 overage on paid plans. Free trials: 14 days Essential/Advanced, 30 days Ultimate (once per store). VIP Tiers, Milestones, Rule engine, POS and cart-drawer redemption start at **Advanced**; checkout-page redemption/API/Hydrogen are **Ultimate**. POS-only or small stores often ask for cheaper access — standard pricing applies, but high-volume/multi-store merchants can ask the Sales team about a custom quota/rate. We can't grant extended free trials beyond the standard ones; for verified high-volume deals, Sales handles custom packages.

### Q29: Can I get a refund / cancel my plan without losing my data? 🔁 recurring
**Frequency:** ~6 sessions
**Standard answer:**
To cancel without deleting your setup: downgrade to **Free/Starter** (Shopify Admin → Settings → Billing, or via Joy's plan page) — your configuration and customers' points are retained, and you can reactivate later. To **pause** earning entirely, put your programs in **Sandbox** and turn notifications off. For a refund, Joy has a **30-day refund policy** after upgrading; per CS process, refunds need CS Leader sign-off — we first understand the goal and try to resolve the issue, then collect the invoice screenshot (Shopify Admin → Settings → Billing → Bills) and escalate. Multi-store discount requests are handled case by case by the team.

## 13. Widget & Loyalty Page

### Q30: The widget isn't showing, is in the wrong position, or overlaps the checkout button (esp. mobile). 🔁 recurring
**Frequency:** ~16 sessions
**Standard answer:**
First confirm it's enabled: **Theme Editor → App embeds → Joy Loyalty - Widget → ON → Save**, then reload. If it shows only after login, that's the "Display after login" setting (intended for logged-in-only). For **position/overlap** — a very common mobile issue where the float button covers the checkout button — use the widget position settings to move it (up/right) or hide it on specific pages/markets. Several merchants found the **Unified** widget overlapped checkout or loaded slowly on mobile and temporarily switched back to **Classic**; if the Unified widget impacts your mobile checkout or load speed, tell us and we'll reposition it or log the performance issue. For pixel-precise placement, share a screenshot + device + page URL and we'll apply a CSS adjustment via collaborator access.

### Q31: How do I hide the widget but still let customers reach the loyalty popup (e.g. via a link or in customer accounts)? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
You can hide the floating widget (turn off the app embed, or hide it per page/market) and instead surface loyalty through (a) a **deep link / button** to the loyalty page or widget (Joy Admin → Settings → General → Deep links), placed in your menu or a custom page, or (b) the **loyalty hub inside Customer Accounts** plus blocks on product/cart pages. If hiding works on desktop but the bubble still shows on mobile, send us the page and we'll fix the mobile rule. Tell us where you want the entry point and we'll provide the correct link/placement.

### Q32: How do I customize the loyalty page / widget — text, colors, images, button icon, layout? 🔁 recurring
**Frequency:** ~12 sessions
**Standard answer:**
Build/edit the loyalty page via **Online Store → Themes → Customize → Add section → Joy Loyalty** (Loyalty page requires Essential+) — arrange blocks (points, ways to earn, rewards, VIP tiers, referral, FAQ/T&Cs). Brand colors/fonts and the global button colors are in **Joy Admin → Settings** (button colors apply across all page sections). For the **Unified** widget launcher button icon/color: if it still shows the default crown after editing, that's a known sync case — tell us the exact icon and hex and we'll apply it. For deeper layout (section widths matching the nav, custom backgrounds, image sizing) share collaborator access + the exact specs and we'll do the CSS. You can self-edit most images/text in the widget and page editors.

## 15. Integrations 🆕 (expanded)

### Q34: Can I auto-issue a coupon via Shopify Flow when a customer hits order/spend conditions (with a delayed start)? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Yes, using **Shopify Flow** (Joy connects at **Joy Admin → Settings → Integrations**). You can build a flow that fires when conditions are met (e.g. 5 orders **and** $200 total spend) to issue an exclusive coupon. A few realities to set expectations: a Flow triggers per-event, so combining "X orders AND $Y spend AND only orders after today" usually needs the flow logic tuned (and may need a tech-assisted setup) — share your exact rule and we'll map it. For a coupon you want to **start later** (e.g. valid only in July) note that a coupon's validity window controls when it can be used, not which orders qualify; if you need orders only counted in a date range, that's a Flow condition. Where Joy can't fully express the logic, we log it as product feedback and suggest the closest workable setup (e.g. issuing the coupon manually to a qualifying list).

### Q35: Does Joy integrate with Klaviyo / Judge.me & review apps / Shopify Flow / POS — and can Klaviyo emails show tier & points? 🔁 recurring
**Frequency:** ~6 sessions
**Standard answer:**
Yes. Email marketing (Klaviyo, Omnisend, AfterShip) — connect under **Joy Admin → Integrations** and, for connection errors, re-check the API key (use a fresh private key and re-paste; if the field won't refresh, re-activate the integration). Review apps (Judge.me, Loox, etc.) can **award points for reviews** (Pro+); you can use Joy's reward instead of a separate review-app coupon to avoid double discounts. **Shopify Flow** (Essential+) powers advanced triggers and subscription/renewal data. **Shopify POS** (Advanced+) lets customers earn/redeem in-store. A dynamic Klaviyo email footer showing the customer's tier and points balance is achievable via the Klaviyo integration's data — we can guide the setup. For data warehousing, full API access is Ultimate.

## 18. Fraud & abuse 🆕

### Q39: A customer used/generated Joy discount codes that weren't officially issued. 🆕 new
**Frequency:** ~1 session
**Standard answer:**
This is treated as a security issue. If an account appears to have generated or used Joy codes it shouldn't have (e.g. suspected-fraud orders), send us the customer email and the order/code details. The team closes the loophole at the source to prevent the pattern recurring.
