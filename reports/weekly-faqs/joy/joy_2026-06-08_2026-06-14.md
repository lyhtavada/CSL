## 1. Getting Started & Setup

### Q1: How do I get started / launch my loyalty program? 🆕 new
**Frequency:** ~10 sessions
**Standard answer:**
Five steps: (1) pick one program type to start — Points is best for most stores; (2) set earning + redeeming under **Joy Admin → Reward Programs**; (3) build a loyalty page via **Online Store → Themes → Customize → Add section → Joy Loyalty**; (4) enable the storefront widget in **Theme Editor → App embeds → Joy Loyalty - Widget → Save**; (5) promote it through your menu, email, and social. A good starter setup: 100-point sign-up reward, 1 point per $1, 100 points = $5 off. Loyalty requires **Shopify Admin → Settings → Customer accounts** to be enabled (Legacy, New, or Shop app login).

### Q2: Can I rename "Points" to something branded? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Yes. Go to **Joy Admin → Settings → General → Custom point label** and rename "Points" to your brand term (e.g., "Beans", "Coins"). The new label updates across the widget and loyalty page.

## 2. Points — Earning

### Q3: Do guests earn points, or only logged-in members? 🆕 new
**Frequency:** ~14 sessions
**Standard answer:**
Customers must have an account to **hold** a points balance. A guest who checks out can have points credited retroactively once they create/log into an account with the same email, but points are tied to the customer account — not to anonymous guest sessions. Make sure **Customer accounts** is enabled in Shopify so customers can sign up and see their balance.

### Q4: Why didn't points get awarded for an order? (COD / manual / draft orders) 🆕 new
**Frequency:** ~12 sessions
**Standard answer:**
Points are awarded when an order reaches the status you've configured (typically **Order paid/fulfilled**). Common reasons points don't appear: the order is still unpaid (COD before collection), it was created as a **draft/manual order**, or the customer wasn't logged in / had no account at purchase time. Check the earning rule's trigger condition under **Joy Admin → Reward Programs → [program] → earn on (paid vs fulfilled)**, and confirm the order actually reached that state. For COD stores, set earning to fire on **fulfillment** rather than payment.

### Q5: Can I set different point amounts per product / exclude products? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
Yes — use point-earning rules to set bonus points or exclusions for specific products/collections. This is configured in the earning program rules (Advanced plan adds the **Rule engine** for more granular conditions). For per-product bonus points, set up a dedicated earning rule scoped to that product/collection.

### Q6: A customer was refunded — do their points get taken back? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Yes. When an order is refunded or cancelled, Joy revokes the points earned on that order automatically, so the customer's balance reflects the reversal. If a refunded order still shows points credited, share the order number and customer email with support to investigate.

## 3. Points — Redemption & Discount Codes

### Q7: A redeemed discount code isn't working at checkout. 🆕 new
**Frequency:** ~7 sessions
**Standard answer:**
First confirm the code is being applied to an eligible cart (minimum spend, product/collection restrictions, one-use rules). Joy-generated codes are standard Shopify discount codes — if a code is rejected, check it hasn't already been used, hasn't expired, and that the cart meets any conditions on the redemption rule. For free-shipping redemptions, confirm the store has shipping rates set for the destination. If the code still fails on a valid cart, escalate with the code + order details.

### Q8: How does free shipping redemption work? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Set up a **Free shipping** redemption option in your redeeming programs; the customer spends points to generate a free-shipping discount code applied at checkout. It only discounts shipping that Shopify actually charges — if a zone has no shipping rate or already-free shipping, there's nothing for the code to discount.

## 4. Migration & Importing Data

### Q9: How do I migrate from another loyalty app (Smile, Rivo, LoyaltyLion, Appstle…)? 🆕 new
**Frequency:** ~11 sessions
**Standard answer:**
Joy migrates Customer Name, Email, Phone, VIP Tier name, Birth date, Customer status, and Point balance (available on Essential/Advanced/Ultimate). There are two paths: **Import** (self-serve, **Customers → Import** CSV — points balance + birthday only) and **Migrate** (assisted — full data including tiers/status/phone). For a full migration: contact support to enable advanced migration, then **Joy Admin → Settings → Migration** → upload your prior platform's export → map tier thresholds → **Save**. Sync takes ~5–6 minutes (up to 20 max). Dedicated guides exist for Smile, Rivo, Yotpo, LoyaltyLion, Stamped, Bon, Appstle, and generic CSV.

### Q10: How do I export my Joy customer / points data? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
You can export customer + points data from the Customers area. For structured analytics or a one-off full export, contact support and specify what fields you need (some larger exports are run on the backend). Note: full API access for programmatic export is an Ultimate-plan feature.

## 5. VIP Tiers

### Q11: How do I set up VIP tiers and tier entry rewards? 🆕 new
**Frequency:** ~9 sessions
**Standard answer:**
VIP Tiers are on **Advanced/Ultimate**, configured at **Joy Admin → Membership → VIP Tiers**. Pick a calculation rule (Points earned, Amount spent, or Number of orders). For a one-time reward when a customer reaches a tier, use **Entry reward**: Membership → VIP Tiers → [tier] → **Entry reward** → Add reward → choose type → Save → **Launch**. For an ongoing perk applied on every qualifying order (e.g. Gold always gets free shipping), use **Tier privileges → Add perks** instead. A common mix-up: an entry-reward coupon "not appearing" usually means it was added under Privileges instead of **Entry reward**, or the tier wasn't re-Launched after editing.

### Q12: My VIP tier counts are wrong — canceled/expired orders are still counted. 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Tier progress is based on the calculation rule you chose (amount spent / points / orders). If canceled or refunded orders appear to still count toward a tier, share the customer email + the orders in question with support so the team can recalculate. For stores that want inactive top-tier customers demoted over time, **Tier assessment** (Ultimate/Plus, amount-spent tiers only) can auto-review and downgrade based on a spend window.

### Q13: Can I manually assign a customer to a special tier (ambassadors, wholesale)? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Yes — create an **Exclusive tier** (Advanced/Ultimate): **Membership → New tier → check "Exclusive tier (Manual only)"** → set rewards/perks → Save. Then assign customers via **Joy Admin → Customers → [name] → Assign to exclusive tier**. Each customer can be in one exclusive tier at a time and won't auto-progress through the normal VIP tiers while in it.

## 6. Referral

### Q14: How does the referral program work and when does the referrer get rewarded? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
The referred friend gets their reward (e.g. a discount) via the referral link; the referrer is rewarded **after the friend's qualifying order completes** (not just on clicking the link). Both parties typically need accounts so rewards can be attributed. If a referral reward didn't fire, confirm the friend completed a qualifying order with the referral cookie/link intact (incognito or a different device/browser breaks attribution). Referral limits scale by plan (Starter 30 / Essential 100 / Advanced+ Unlimited).

## 7. Store Credit vs Points

### Q15: What's the difference between store credit and points? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Points are Joy's loyalty currency that customers redeem for discount rewards you define. Shopify **store credit** is a separate native balance. Joy's redemptions generate discount codes rather than topping up Shopify store credit. If a merchant specifically wants native store-credit payouts, log it as product feedback and clarify their goal — most reward use cases are fully covered by points → discount redemptions.

## 8. Customer Accounts & Duplicate / Merge

### Q16: A customer has duplicate accounts / points are split across two profiles. 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
This usually happens with Customer Accounts 2.0 when a customer checks out under two emails or a guest checkout splits from their account. Collect both customer emails and the correct surviving account; support can merge the profiles and consolidate the point balance. Encourage customers to always log in before checkout to avoid splits.

## 9. Birthday Rewards

### Q17: How do birthday rewards work and why didn't a customer get theirs? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Birthday rewards fire on the customer's stored birth date. The birthday must be captured in the correct format and exist before the date passes — a birthday added after the date won't backfill that year. Confirm the customer entered their birthday (and the import used MM/DD format). Enable the birthday email under **Joy Admin → Settings → Notifications**.

## 10. Transaction / Order Limits

### Q18: I hit a transaction limit but only have a few orders — why? Can I get unlimited? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
A **transaction in Joy is not the same as a Shopify order** — 1 transaction = 1 loyalty activity (earning, redeeming, referral, etc.), so a single order can create multiple transactions and the count climbs faster than your order count. If you've hit the transaction limit but are still within your plan's order quota, support can enable **Unlimited transactions** for verified stores — share your store URL and we'll escalate to the team.

## 11. Pricing, Plans & Refunds

### Q19: How much does Joy cost / what's on each plan? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
Starter $0 (250 free orders/mo), Essential $29/mo (500 orders, $15/100 overage), Advanced $129/mo (2,000 orders, $10/100), Ultimate $499/mo (7,000 orders, $5/100). Free trials: 14 days Essential/Advanced, 30 days Ultimate (once per store). VIP Tiers, Rule engine, POS, cart-drawer redemption and Milestones start at **Advanced**; checkout extensions/API/Hydrogen are **Ultimate**. Merchants subscribed before Feb 9, 2026 keep grandfathered pricing.

### Q20: Can I get a refund? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Joy has a **30-day refund policy** after upgrading if the merchant isn't satisfied. Per CS process, CS is not authorized to approve refunds without CS Leader sign-off: understand the merchant's goal, try to resolve the underlying issue first, then collect the invoice screenshot (app name, billing cycle, amount; from Shopify Admin → Settings → Billing → Bills), guide them to downgrade to Free, and escalate to the CS Leader.

## 12. Widget & Loyalty Page

### Q21: The loyalty widget isn't showing / is in the wrong position / overlaps another button. 🆕 new
**Frequency:** ~9 sessions
**Standard answer:**
First confirm the widget is enabled: **Theme Editor → App embeds → Joy Loyalty - Widget → ON → Save**, then reload the storefront. For position/overlap with other app buttons (cart drawer, chat, scroll-to-top), use the widget's position settings to move it or hide it on specific pages. For pixel-precise placement the settings can't cover, collect a screenshot + device + the page URL and escalate to the team for a CSS adjustment.

### Q22: How do I customize the loyalty page / what customers see? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Build and edit the dedicated loyalty page via **Online Store → Themes → Customize → Add section → Joy Loyalty** (Loyalty page requires Essential+). You can arrange blocks (points balance, ways to earn, rewards, VIP tiers, referral). For storefront copy in other languages, use Joy's translation settings (auto-translate on Advanced+).

## 13. Translations

### Q23: How do I translate the widget / loyalty content into another language? 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Use Joy's translation settings to localize widget and loyalty-page text. Basic multi-language is on Essential; **auto-translate** is on Advanced/Ultimate. Set your store's languages in Shopify, then provide/adjust the translated strings in Joy so the widget matches the storefront language.

## 14. Integrations

### Q24: Does Joy integrate with Klaviyo / Omnisend / Shopify Flow / review apps / POS? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Yes. Joy supports email-marketing integrations (Klaviyo, Omnisend), **Shopify Flow** (Essential+), review apps (award points for reviews — judge.me, etc.), and **Shopify POS** (Advanced+) so customers earn/redeem in-store. Each integration is enabled in the relevant **Joy Admin → Integrations** section; for Klaviyo/Omnisend you connect the account and map events. For data warehousing/Looker-style exports, full API access is Ultimate.
