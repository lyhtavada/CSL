## 1. AI Agent — Setup & Training

### Q1: How do I set up and train the AI agent for the first time? 🆕 new
**Frequency:** ~18 sessions
**Standard answer:**
Advanced AI features (Store Instructions, Scenarios, Q&A, expanded Data Sources) require a **paid plan**. Steps: (1) **AI agent → Training data → Manage** to add Products, Collections, Discounts, Markets, FAQs, and Custom knowledge; (2) **AI agent → Instructions → Manage** to set the AI's role, tone, and behaviours; (3) **AI agent → Test AI → Test now** to check responses before going live (the Test feature does **not** consume your AI-replies quota). Many "generic AI" complaints are because Instructions were never configured — set the Role and Behaviours first.

### Q2: How many products can the AI train on? 🆕 new
**Frequency:** ~9 sessions
**Standard answer:**
Product sync limits by plan: **Free 100, Basic 500, Pro 8,000, Plus 20,000**. If you exceed the limit, Chatty trains on the most recently updated products first. Stores with more than 5,000 products need manual backend setup — contact support. To pull the latest catalog: **AI agent → Training data → Manage → Products → Sync products** (note: real-time stock quantity is not auto-updated).

### Q3: How do I create a custom scenario / make the AI follow a specific flow? 🆕 new
**Frequency:** ~7 sessions
**Standard answer:**
Go to **AI agent → Instructions → Manage → Assistant skills → Custom scenarios → + Add scenario**, then set a scenario name (the topic customers mention) and instructions (the flow you want). Use it for returns, order lookups, discount requests, sizing, or any process you want handled consistently. If you hit the scenario character limit, contact support — the limit can't be raised but the team can help shorten the text.

### Q4: Can I upload files (PDF/CSV) to train the AI? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Yes — **AI agent → Training data → Manage → Custom knowledge → Add data → Files**. Supported: PDF, CSV, TXT, JSON, **max 2MB per file**. CSV is recommended for structured Q&A and product catalogs. For external website content, submit individual page URLs (the domain alone only trains on the homepage); Shopify policy pages (shipping, returns, privacy, etc.) are synced automatically.

### Q5: Can I add more AI conversations — I ran out? 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
Each plan (except Plus) has a limited number of AI replies/conversations. If a merchant runs out mid-cycle, CS can add a bonus allotment via the internal tool, or the merchant can upgrade for a higher ongoing limit. Confirm the merchant's plan and current usage first, then top up or recommend the right plan.

## 2. Inbox Cross-Contamination Bug ⚠️

### Q9: Messages/replies are showing up in the wrong customer's chat (mixed conversations, blank inbox). 🆕 new
**Frequency:** ~15 sessions
**Standard answer:**
This is a recognized defect, not a setting the merchant can fix — collect the **conversation/session IDs**, affected customer emails, a screenshot, and the approximate timestamps, then escalate to the dev team immediately. Tell the merchant: "Thanks for flagging — this is a system-side issue our team is investigating; I've logged your conversation details so they can trace it." Do not advise cache-clearing or reinstalls as a fix. **Internal note for Liz:** high volume across multiple languages this week — worth raising with product/dev as a priority bug, not one-off.

## 3. AI — Wrong / Hallucinated Responses

### Q6: The AI is making up products, links, or info that doesn't exist. 🆕 new
**Frequency:** ~8 sessions
**Standard answer:**
This usually means the AI matched the query against incorrect backend data (e.g. a bad collection lookup). Collect the **conversation ID** and send it to support — the team adds an instruction hook to correct the response logic. As prevention, make product descriptions detailed and unambiguous, and add the correct facts to **Custom knowledge / FAQs**. For wrong-catalog cases (products from another store), same path: conversation ID → support.

### Q7: The AI answers in the wrong language / switches language mid-chat. 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
(1) **Settings → Translation** → confirm the correct default language; (2) **AI agent → Instructions → Manage → General instructions → Behaviours** → add: "Always respond in the same language as the customer." For persistent issues on very short/ambiguous messages, send the conversation ID to support for an explicit per-store instruction. Real-time agent↔customer translation in the inbox requires **Pro or Plus**.

### Q8: The AI sounds generic / doesn't match our brand voice. 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Set tone at **AI agent → Instructions → Manage → General instructions → Communication style / Behaviours** (e.g. "Friendly assistant for [Brand], warm casual language", "Keep responses under 3 sentences"). You can also set the bot **Role**, welcome message, and avatar in General instructions. Changes apply immediately — no republish needed. Most "generic" complaints are stores that never filled in Instructions.

## 4. Human Handover

### Q10: The AI isn't handing conversations off to a human agent. 🆕 new
**Frequency:** ~7 sessions
**Standard answer:**
Check **AI agent → Instructions → Manage → Assistant skills → Customer support skills → Human handover**: (1) **Auto triggers** — "Customer asks to talk to a person" is always on; "AI cannot answer", "Customer repeats question", and "Negative sentiment" are toggles, so enable the relevant one; (2) **Destination** — confirm "Transfer to human in Chatty inbox" is selected; (3) **Offline behavior** — if "Allow only outside business hours / all the time" is set, the AI keeps replying instead of handing off (expected). If configured correctly but the conversation stays "Unassigned," check agent routing in **Automation settings**, then escalate with conversation details.

### Q11: The AI keeps replying after a human took over / handover routing to email. 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Choose the right destination under Human handover: **Transfer to inbox** (agent joins the same chat), **Collect info & follow up by email** (no live agents — AI collects details and emails your team, with a response-time promise), or **Show contact methods**. If the AI resumes after a human joined, confirm the offline/"let AI keep replying" setting isn't set to "Allow all the time," and check the conversation is actually assigned. Persisting issues → conversation ID to support.

## 5. Widget / Chatbox

### Q12: The chat widget isn't showing on my store. 🆕 new
**Frequency:** ~8 sessions
**Standard answer:**
Sequential check: (1) if it worked before, toggle the **App Embed** off/on — **Shopify Admin → Online Store → Themes → Customize → App Embeds → Chatty** → OFF → Save → ON → Save, reload storefront; (2) confirm **App Embed ON** and **Chatbox → General → Blocks → Live chat** toggled on. There is no separate "Chatbox status" toggle — don't chase theme-compatibility or cache steps. If still missing, use the collaborator-access flow and escalate to TS with store URL, theme name, and steps tried.

### Q13: The chat button overlaps my Add-to-Cart / another app button (mobile or desktop). 🆕 new
**Frequency:** ~7 sessions
**Standard answer:**
Collect first: which element is blocked, which page(s), device, and a screenshot. Then offer the self-serve fix — **Chatbox → Appearance → Chatbox button → Position**: drag the dot or pick an **Anchor** + **Horizontal/Vertical offset** (0–45%) to move it clear. Alternatively hide it on specific pages via **Chatbox → Advanced → Display settings → Display pages**. Only escalate to TS for Custom CSS if the position control can't cover it (beyond 45%, pixel-precise).

### Q14: How do I change the widget colour, icon, name, or avatar? 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Widget appearance (colour, button shape/icon) is under **Chatbox → Appearance**. The AI bot's name, welcome message, and avatar are set under **AI agent → Instructions → Manage → General instructions** (Role / Behaviours). Avatar upload not working? Use Chrome, image must be JPG/PNG under 2MB, try incognito; if it still fails, escalate with store URL + file details.

## 6. FAQ Page

### Q15: My FAQ page / categories aren't showing (only some appear). 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
(1) Enable it: **FAQs → FAQs page → Display FAQs page → ON** and Save; (2) on the same page, turn on **"Embed app to your theme"** if Off; (3) add the FAQ page URL (default `/pages/avada-faqs`) to your nav via **Shopify Admin → Content → Menus**. If FAQ formatting/bullets render broken, that's a known rich-text issue — avoid pasting HTML, use plain text, and the team can apply a CSS fix via a TS ticket. Category icons are locked on new installs; CS can unlock them via the internal tool with your store URL.

## 7. Email Channel & Notifications

### Q16: My email channel is stuck "unverified" / AI isn't replying to emails. 🆕 new
**Frequency:** ~6 sessions
**Standard answer:**
Confirm the AI email channel is enabled: **AI agent → Instructions → Manage → General instructions → AI channels → Email**. The AI auto-replies after 5 minutes if no agent responds (genuine customer inquiries only — not spam/system mail). If verification is stuck or configured correctly but the AI still won't reply, collect the email address + forwarding setup and escalate to the dev team.

### Q17: I'm not getting notifications for new chats (no email/push). 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Check the notification settings in the app and that the agent's email/device is set to receive alerts. The **Chatty mobile app is a PWA** (installed from the browser/your account), not a native App Store app — set it up and enable push there for on-the-go alerts. If notifications are configured and still missing, collect details and escalate.

## 8. Pricing, Plans & Refunds

### Q18: How much is Chatty / what's on each plan? 🆕 new
**Frequency:** ~7 sessions
**Standard answer:**
Plans gate AI features and limits — Free is limited (100 products, capped AI replies); paid plans (Basic, Pro, Plus) raise product sync (500 / 8,000 / 20,000) and AI conversation limits, and unlock Store Instructions, Scenarios, Q&A, real-time inbox translation (Pro+), and product recommendations (Pro+). Confirm the merchant's current plan in Shopify, then map their need (product count, AI volume, translation) to the right tier. For exact prices, refer to the in-app billing page / chatty pricing page.

### Q19: I want a refund / was charged unexpectedly. 🆕 new
**Frequency:** ~5 sessions
**Standard answer:**
Per CS process, CS is **not** authorized to approve refunds without CS Leader sign-off. For a simple cancellation: guide the merchant to downgrade to Free, collect the invoice screenshot (app name, billing cycle, amount; from Shopify Admin → Settings → Billing → Bills) and whether it's paid, then create a Trello card for the CS Leader. For bug/dissatisfaction cases, try to resolve the underlying issue first; if unresolved, escalate. Refunds take 7–10 business days once approved. Charged after uninstalling? Shopify still bills the started cycle but refunds unused days.

## 9. Order Tracking, Translation & Channels

### Q20: How do I set up order tracking in the chatbox? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Chatty's order-tracking page lets customers look up their order/shipment in the widget (integrates with tracking providers such as 17track). Enable it in the chatbox blocks/settings, then customers reach it via the widget's tracking section (`ChattyJS.openWidget('#chatty-tracking')` can deep-link to it). Confirm the integration is connected so live tracking data shows.

### Q21: How do I connect WhatsApp / Messenger — mine is stuck "pending"? 🆕 new
**Frequency:** ~4 sessions
**Standard answer:**
Connect channels under the channel/integration settings. A connection stuck "pending" usually means the authorization didn't complete or the account needs re-permissioning — have the merchant disconnect and reconnect, ensuring they approve all requested permissions on the provider side. If it stays pending after a clean reconnect, collect the store URL + channel and escalate to the team.

### Q22: How does auto-translate work / I hit the translation limit. 🆕 new
**Frequency:** ~3 sessions
**Standard answer:**
Chatty supports translating chatbox/FAQ content into multiple languages; the number of languages is limited by plan. Real-time agent↔customer message translation in the inbox requires **Pro or Plus**. If a merchant hit a language cap, confirm their plan and recommend the tier that covers the languages they need.
